# Notion Export — 省エネ提案を Notion へ出力する

このリポジトリ（`kizuna-proposal`）の提案資料を、構造化した Notion ページとして
作成・更新するためのスキルです。Notion MCP サーバーのツールを利用します。

## いつ使うか
- 「提案内容を Notion にまとめて」「Notion に出力／連携して」と頼まれたとき
- 提案のスナップショットを Notion 上に記録・共有したいとき
- スライドやナレーションを更新したあと、Notion 側を同期したいとき

## 前提（連携が動く条件）
1. Notion MCP サーバーが接続済みであること。
   利用するツール（論理名）:
   - `notion-search` … 出力先の親ページ／既存ページを探す
   - `notion-fetch` … 既存ページの内容・ID を取得する
   - `notion-create-pages` … 新規ページを作成する
   - `notion-update-page` … 既存ページを更新する
   これらが見つからない場合は `ToolSearch` で `notion` を検索して読み込む。
2. 接続が無い／権限が無い場合は、その旨をユーザーに伝えて中断する
   （勝手に別の場所へ書き込まない）。

## 出力元データ
- スライド: `index.html`（`<div class="eyebrow">…</div><h2 class="title">…</h2>` が各スライドの見出し。
  本文は同セクション内の `.card` / `<p>` / 表）。全8スライド。
- ナレーション: `mp3gen_kizuna.py` の `NARRATIONS` 配列（スライド順、日本語の読み上げ原稿）。
- タイトル: `index.html` の `<title>`（例: 「KIZUNA HOME様 省エネルギー・脱炭素化 ご提案書」）。

## 手順
1. **出力先を決める**
   - `notion-search` で出力先（親ページ or 既存の提案ページ）を探す。
   - 既存ページが見つかれば ID を控え「更新」、無ければ「新規作成」。
   - 出力先が曖昧なら、`AskUserQuestion` で親ページを確認してから書き込む。
2. **データ抽出**
   - `index.html` から 8スライド分の「英語ラベル(eyebrow) / タイトル / 本文」を取り出す。
   - `mp3gen_kizuna.py` から対応するナレーションを取り出す。
   - スライドとナレーションを順番（1〜8）で対応付ける。
3. **ページ構成を作る**（下記「ページ構成テンプレート」に従う）。
4. **書き込み**
   - 新規: `notion-create-pages`（親 = 手順1で決めたページ）。
   - 更新: `notion-update-page`（既存ページの本文を置き換え／追記）。
5. **報告**
   - 作成・更新した Notion ページの URL とページIDをユーザーに返す。

```
# KIZUNA HOME様 省エネルギー・脱炭素化 ご提案書
（出力日: YYYY-MM-DD / 出力元: kizuna-proposal リポジトリ）

## サマリー
- 年間電気代: 約700万円 → A 277万円/B 322万円 削減（満床時 最大558万円）
- 投資回収: A 6.0年 / B 7.6年（満床達成で4年台）
- 補助金見込み: 約496万円 ／ 申請締切 10/7

## スライド別 内容
（各スライドを toggle もしくは見出しで）
### 01. <タイトル>
本文の要点（箇条書き）
> ナレーション原稿（引用ブロック）
…（08 まで）

## プラン比較（A/B）
（index.html の比較表を Notion テーブルに）

## スケジュール / 次のステップ
（SCHEDULE スライドの内容）
```

## 注意
- 数値・固有名詞（金額、kW、kWh、補助金額、連絡先 043-261-8033 / t-terasaki@syscomnet.co.jp）は
  原文どおり正確に転記する。要約で値を変えない。
- 既存ページを上書きする前に、内容が想定どおりか（提案ページであること）を `notion-fetch` で確認する。
  自分が作っていない無関係なページには書き込まない。
